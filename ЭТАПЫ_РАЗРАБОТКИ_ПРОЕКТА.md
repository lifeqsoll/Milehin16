# Этапы разработки проекта

## 1. Постановка задачи и сбор материалов
- Определена цель: создать одностраничный сайт-экскурсию о И. Ф. Милёхине для публикации на GitHub Pages.
- Собраны исходные материалы проекта: документы `docx`, презентации `pptx`, дополнительные фотографии.
- Зафиксированы ключевые требования: главный зал, залы с полноэкранным режимом, фотоархив, ключевые даты, мобильная адаптация.

## 2. Подготовка контента
- Выполнен анализ текстов из документов и презентаций.
- Выделены факты для тематических залов: детство, война, награды, Пост №1, наследие.
- Подобраны и структурированы фотографии, включая дополнительные изображения из папки `фото`.
- Сформированы подписи и расширенные текстовые описания к материалам.

## 3. Проектирование структуры сайта
- Сформирована архитектура одной страницы в `index.html`.
- Спроектированы основные блоки:
  - главный зал (вступление);
  - боковое меню выбора залов;
  - полноэкранный режим каждого зала;
  - раздел "Ключевые даты";
  - раздел "Фотоархив".
- Определен сценарий навигации между залами: предыдущий, главный, следующий.

## 4. Базовая реализация интерфейса
- Реализована HTML-структура и стили CSS.
- Настроена цветовая схема в бело-бежевой гамме с красными акцентами.
- Добавлен визуальный отклик кнопок и карточек (hover/active состояния).
- Реализовано плавное открытие/закрытие бокового меню и полноэкранных залов.

## 5. Интерактивная логика на JavaScript
- Реализована динамическая отрисовка залов и карточек из массива данных.
- Добавлена логика переключения залов в полноэкранном режиме.
- Реализован фотоархив: перелистывание, миниатюры, счетчик, подписи.
- Добавлена поддержка клавиатуры и свайпов на мобильных устройствах.
- Реализовано раскрытие/сворачивание блока миниатюр галереи с анимацией.

## 6. Оптимизация контента и удобства чтения
- Тексты в залах расширены и разбиты на удобные абзацы.
- Улучшено соответствие фотографий тематике каждого зала.
- Исправлены проблемы читаемости (контраст, фон, рамки, отступы).
- Исключены лишние элементы интерфейса (например, дублирующая кнопка закрытия зала).

## 7. Адаптация под мобильные устройства
- Настроены медиа-запросы для экранов планшетов и смартфонов.
- Организован мобильный порядок контента: фото -> текст -> фото -> текст.
- Обеспечена вертикальная прокрутка текста в карточках.
- Улучшена навигация в фотоархиве для сенсорного управления.

## 8. Тестирование и исправления
- Проведена серия ручных проверок пользовательских сценариев:
  - открытие/закрытие меню и залов;
  - переходы между залами;
  - работа фотоархива;
  - работа интерфейса на мобильных экранах.
- Исправлены найденные ошибки в логике и стилях.
- Выполнена финальная проверка на корректность отображения и стабильность работы.

## 9. Результат
- Готов одностраничный сайт-экскурсия в `index.html`.
- Проект соответствует целям: информативный, визуально цельный, адаптивный и готовый к публикации на GitHub Pages.
